adb和fastboot

这里会列出在刷机过程中比较常用的adb和fastboot命令

adb全称Android Debug Bridge,在进入安卓系统后方可使用,部分rec也是可以用adb的。

fastboot是用于设备刷写镜像的一个模式,一般来说刷个root或线刷用这个就行。
前往谷歌官方文档查看

adb reboot <重启到模式> 括号处填入需要进入的模式即可在开机状态下立刻进入,比如 recovery 进入rec,bootloader进入fastboot。

adb install <安装包路径> 括号处填入安装包路径即可把安装包推送到手机并安装这个app。

adb uninstall <包名> 括号处填入需要卸载的应用包名即可卸载这个app。

adb devices 查看 adb 连接设备。

adb shell 进入终端,这里的终端和mt管理器的一样,也具有危险性,不要执行恶意代码！

fastboot devices 查看已连接的fastboot设备。

fastboot flash <分区><镜像路径> 分区填入需要刷入的分区,比如刷root就要刷boot分区,路径填入文件路径,在win11可以右键直接复制文件路径。

fastboot reboot 从fastboot重启手机。

fastboot oem lock 重新锁上bootloader。

fastboot boot <镜像路径>用来临时启动twrp或boot,只会临时启动,不会刷写到任何分区,但是联发科大部分机型都不支持！

fastboot getvar current-slot 查看当前a/b槽位

fastboot set_active <要切换的插槽> 切换当前的槽位,输入a/b。

注意事项:临时启动这个命令大部分联发科机型都是不支持的！联发科机型请不要乱刷东西,联发科是极其容易变转的！这里只会记录部分比较常用的命令,如有错误可以联系我更正,会持续更新......

Root教程

这里会较为详细的讲述root的教程

首先得了解root是什么
root,是一个权限,手机中的超级用户权限,具有读写根目录的权限,可以操作系统进程,注入内存等操作。
前往百度百科查看

Root方式选择:

magisk:这是为安卓深度定制的su管理软件,因为其强大的影响力,深受机佬的喜爱,也诞生了许多的版本分支,其中有阿尔法(Alpha)、德尔塔(Delta/Kitsune), 除此之外还有各路大神制作针对某个机型的magisk版本,下面会讲到如何选择magisk版本。
前往百度百科查看

Kernel su:这个root方式是近些年兴起的一种,他的工作方式和magisk非常不同,ksu的工作方式是在内核操作,所以非常难被检测到,因其强大的隐藏性,也是 不少人的选择,但是相对magsik来说,有一个非常致命的缺点,就是模块的兼容性非常差,挂载也有问题,所以一般来说是日用选择ksu,而极客更偏向于magisk,也许作者以后会完善。
前往Kernel su官网查看

Magisk安装教程:
1.准备好你现在系统版本对应的boot,可从线刷包/卡刷包里面提取,小米设备可以去Xiaomirom查看
2.打开你的magisk,选择安装,选择修补,然后选择你提取出来的boot,完成后把得出来的boot传到电脑上
3.手机进入fastboot模式,开机按住音量-和电源键即可进入,也可以开机状态下使用adb reboot bootloader命令进入
4.连接电脑,打开你的adb,输入fastboot flash boot 修补boot的路径,等待刷入完成即可,如果不懂请查看adb&fastboot常用命令

注意,在新机型中需要修补的应该是init_boot,例如小米13系列,具体请查询!

Kernel su安装方式:
1.确认你手机的内核版本,在手机的全部参数滑到底下即可看到,需要版本大于5.xx.xxx,否则请使用面具
2.到Ksu的Releases页面找到你对应的内核版本
3.下载文件后解压出里面的boot镜像
4.手机进入fastboot模式,开机按住音量-和电源键即可进入,也可以开机状态下使用adb reboot bootloader命令进入
5.连接电脑,打开你的adb,输入fastboot flash boot 解压boot的路径,等待刷入完成即可,如果不懂请查看adb&fastboot常用命令

注意:联发科机型必须谨慎操作,不然各种姿势变砖和丢数据,天玑1200以下机型magisk无法直接更新,需要自己修补后重新刷入!

注意事项


这里会列出一些需要注意的东西
1.联发科机型必须要注意的你的内核版本刷twrp,不然会变砖!
2.联发科天玑1200以下机型不能使用magisk自带的直接更新来更新面具版本,因为有分区保护,需要进入fastboot重刷
3.联发科天玑8200以上出厂的安卓13新机不能直接刷kernel su的通用内核gki,请先用面具root后使用ak3卡刷
4.联发科不要随意使用更改system下app的magisk模块,稍有差池你的data就不保啦

隐藏环境-德尔塔

目前不存在完美隐藏,能做的也只有尽量去隐藏,遇到过不了的不要太执着!


德尔塔最新的面具也可以过很多检测,也相对简单,非常适合小白和懒癌患者
德尔塔无需安装shamiko或者zmod这一类的隐藏模块

黑名单设置：
打开Magisk Hide,不需要开启强制SuLite,把需要隐藏的应用在下方的配置列表勾选即可

白名单设置：
打开Magisk Hide,开启强制SuLite,把需要root权限的应用在下方的配置列表勾选即可

自带的Zygisk已经更新为Zygisk Next,所以不需要再额外安装,请放心使用,如果还是过不了某些应用的检测还是请试试关闭自带的zygisk刷入最新 zygisk next模块。检测工具推荐使用momo和native test

最新版德尔塔面具
最新版Zygisk next
Native Test v12
momo (以后补上)
native test v15 (以后补上)

隐藏环境-阿尔法

目前不存在完美隐藏,能做的也只有尽量去隐藏,遇到过不了的不要太执着!


阿尔法面具的隐藏会比较麻烦,但也很容易
阿尔法面具不要开启自带的zygisk,必须安装Zygisk Next,虽然模块列表显示未开启zygisk,模块停用,但实际上模块是正常运行的,请放心使用

1.安装Zygisk Next
2.关闭面具自带的Zygisk
3.刷入最新的Shamiko模块
4.设定好你需要的Shamiko运行模式,白名单或是黑名单
5.重启

你也可以不选择shamiko,可以选择zmod

安装后shamiko把面具的权限也杀掉了?没关系,shamiko的开关是动态的,你只需要借助爱玩机这一类的工具去取消模块的激活,面具刷入模块或授权后重启打开就行了, 嫌麻烦的我下面会提供一个快捷的shamiko开关,他可以添加磁贴到控制中心
如果你选择了zmod,那么记得给你的面具隐藏包名,在面具的设置里即可找到,隐藏后zmod就不会杀掉magisk的权限了,zmod只有白名单模式,没有黑名单模式
